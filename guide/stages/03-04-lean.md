# Stage 3+4 — Brand + experience (lean)

## Goal

Capture enough brand and experience direction to guide implementation — without the full workshop.

Three questions is the target. Enough to prevent invention during build, not a complete brand system.

## Ask these questions, one at a time

1. "How should this feel to use — and what should it definitely not feel like?"
2. "Is there a product or interface whose character feels close to what you want? Even a rough reference helps."
3. "What's the main thing a user does in the first version — and should it feel more like a focused tool or an open workspace?"

If any answer is already clear from Stage 2, skip it and note the answer as assumed.

## Output from this stage

Brief notes for:

- `docs/brand.md` — essence, desired feeling, what it is not, one visual direction signal
- `docs/design.md` — experience shape, first user flow, interaction style

These do not need to be complete documents. Enough direction to avoid guessing during implementation.

## Completion checklist

Before moving to Stage 4 (Technical direction), confirm you have all of these:

- Feeling direction captured (how it should feel, what it should not feel like)
- At least one reference or clear rejection noted
- Experience shape decided (tool vs workspace, focused vs open)
- Responsive emphasis noted — even a guess

## Stage checkpoint

When the completion checklist is satisfied:

1. Show the lean progress indicator with Stages 1–3 marked ✓ and Stage 4 marked →.
2. Write or update `decisions/kickoff-checkpoint.md` with the current stage, decisions, assumptions, and open questions.
3. Ask: "Ready to continue to Stage 4 — Technical direction, or would you like to pause here?"

If the user pauses, let them know the checkpoint has been saved to `decisions/kickoff-checkpoint.md` — starting a new `/project-kickoff` session will pick up where they left off automatically.
