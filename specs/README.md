# Specs

This folder contains product and feature specifications.

## File naming

Use:

`YYYY-MM-DD-feature-name.md`

Examples:

- `2026-05-10-mvp.md`
- `2026-05-14-user-onboarding.md`
- `2026-05-18-newsletter-preferences.md`

## When to create a spec

Create a spec for:

- a new product
- a major feature
- a meaningful redesign
- a workflow that affects multiple parts of the product
- any work where product behaviour, technical structure, or scope could otherwise become unclear

Use `docs/spec.md` as the standard structure.

## Suggested workflow

1. Read the core files in `/docs`
2. Create or update the relevant spec
3. Review assumptions and open questions
4. Build from the spec
5. Update the spec if the product decision changes
